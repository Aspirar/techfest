Problem Statement

Participants individually compete in two different rounds of the challenge. In each round, participants are assigned an embedded task. Participants need to bring their own Atmel series (only) microcontroller and related hardware or an Arduino development board and code in the Arduino IDE (only). Participants are provided the required hardware to implement the specified task in each round (except the microcontroller and related hardware).

Round 1

  • A simple task is given to check the understanding of programming concepts.
  • The participant has to raise his/her hand in order to get tagged with the timestamp after writing the code in order to tabulate their position in the ranking table.
  • Now, the participant is provided with the required hardware. He/she gets a single chance to showcase the output on the hardware.
  • If the participant fails to showcase the output on the hardware on the first try, he/she is disqualified immediately.
  • Only a maximum of 30% of the participants are filtered into next round based on their ranking.
  • If the expected number of participants are unable to qualify for the next round, then the whole procedure is repeated with participants allowed to have a second try on the hardware. Participants in this case, who have accomplished the task in the first try itself, move to the next round immediately.

Round 2

  • This round will test the knowledge of the participants based on simple communication protocols using a module and/or other sensors.
  • Participants are not allowed to use libraries other than those that are in-built in the Arduino IDE.
  • Information about the module will be provided before 10 days to fest on the official website of the fest and in the android app.
  • Top two performers based on their rank attained by a similar procedure in round 1 win the prizes.

Rules

  • Participants are not allowed to use libraries other than those that are in-built in the Arduino IDE.
  • He/she will get a position on the ranking table only when the code successfully gets implemented on the hardware.
  • The rank will be queued based on the timestamp provided to the participants.
  • The participant has to explain his/her code if asked for and go through different test cases if needed.
  • Team will have no access to cellphones.
  • Taking help of any kind will result in immediate disqualification.
